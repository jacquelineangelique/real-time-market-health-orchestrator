# Solution Architecture

## Overview

The Real-Time Market Health Orchestrator was designed as a lightweight data integration and business intelligence solution that simulates modern production analytics workflows.

The architecture combines API extraction, ETL processing, dimensional modeling, and Tableau visualization into a single analytical pipeline.

---

```text

        ▼
```

---

## Components

### Extraction Layer

Responsible for retrieving historical market data from external APIs.

Features include:

* Retry logic
* Exponential backoff
* Timeout handling
* Rate-limit handling
* Error logging

---

### Raw Data Layer

Stores source API responses in JSON format.

Benefits:

* Reproducibility
* Auditability
* Troubleshooting
* Source validation

---

### Transformation Layer

Converts raw JSON into a standardized analytical dataset.

Responsibilities include:

* JSON flattening
* Data normalization
* ISO-8601 timestamp conversion
* Daily variance calculations
* Base 100 indexing

---

### Star Schema Modeling Layer

Creates Tableau-ready dimensional tables.

Fact Table:

```text
fact_market_rates
```

Dimensions:

```text
dim_date
dim_asset
dim_source
```

Benefits:

* Simplified reporting
* Improved Tableau performance
* Dimensional analytics
* Scalable design

---

### Visualization Layer

Tableau dashboards provide:

* Market Health Overview
* Volatility & Risk Monitoring

The dashboards consume dimensional tables generated by the ETL pipeline.

---

## Design Principles

The solution was designed using:

* Separation of concerns
* Reusable ETL components
* Reproducible processing
* Dimensional modeling best practices
* Business-user accessibility
